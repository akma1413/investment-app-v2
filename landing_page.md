
# Hypo 랜딩페이지 기획서

## 1. 핵심 문제 정의 (Problem Statement)

### 1.1 투자자들이 겪는 근본적인 고통

**"왜 우리는 투자할 때 불안한가?"**

개인 투자자들의 불안은 단순히 "돈을 잃을까봐"가 아니다. 진짜 문제는 더 깊은 곳에 있다:

| 증상                    | 근본 원인                                        |
| ----------------------- | ------------------------------------------------ |
| 주가가 떨어지면 패닉셀  | **왜 샀는지 기억이 안 남**                 |
| 주가가 오르면 FOMO 매수 | **매수 기준이 없음**                       |
| 뉴스에 일희일비         | **이 뉴스가 내 투자에 어떤 의미인지 모름** |
| 언제 팔아야 할지 모름   | **매도 기준을 세운 적이 없음**             |
| 밤에 잠이 안 옴         | **내 투자가 맞는지 확인할 방법이 없음**    |

### 1.2 기존 솔루션의 한계

```
❌ 증권사 앱: 매매 도구일 뿐, 투자 "판단"을 돕지 않음
❌ 뉴스/리서치: 정보 과잉, 내 포지션과 연결 안 됨
❌ 유튜브/커뮤니티: 남의 의견, 내 기준이 아님
❌ 로보어드바이저: 블랙박스, 왜 그런 결정인지 모름
```

### 1.3 우리가 발견한 인사이트

> **"불안함의 본질은 '기준의 부재'다."**

전문 투자자와 개인 투자자의 차이:

- 전문가는 **가설**을 세우고, 그 가설이 맞는지 **검증**하며 투자한다
- 개인은 **감**으로 사고, **감정**으로 대응한다

**우리의 핵심 통찰:**

```
투자 불안 = 기준 없음 + 검증 불가 + 대응 방법 없음

Hypo의 해결책:
✅ 가설 설정 → 기준 생김
✅ 워치포인트 → 검증 가능
✅ 이벤트 대응 → 행동 방향 명확
```

## 2. 가치 제안 (Value Proposition)

### 2.1 한 문장 정의

> **"Hypo는 당신의 투자 가설을 세우고, 검증하고, 대응하게 도와주는 논리 기반 투자 어시스턴트입니다."**

### 2.2 Before → After 프레임

| Before (기존)                  | After (Hypo 사용 후)              |
| ------------------------------ | --------------------------------- |
| "왜 이 주식 샀더라..."         | "이 가설이 맞으면 오른다"         |
| "이 뉴스 좋은 거야 나쁜 거야?" | "내 워치포인트 2번에 해당하네"    |
| "팔아야 하나 말아야 하나"      | "가설이 깨지면 팔고, 아니면 홀드" |
| "밤에 잠이 안 와"              | "로직 건강도 80%, 안심해도 됨"    |

### 2.3 핵심 가치 3가지

```
1️⃣ 가설 기반 투자 (Hypothesis-Driven)
   → 모든 투자에 명확한 "왜"가 생김
   → 감정이 아닌 논리로 결정

2️⃣ 실시간 검증 (Watchpoint System)
   → 내 가설이 맞는지 계속 체크
   → 뉴스/이벤트가 내 투자와 어떻게 연결되는지 자동 분석

3️⃣ 행동 가이드 (Action Framework)
   → 이벤트 발생 시 Buy/Hold/Sell 판단 기준 제공
   → 로직 건강도로 포지션 상태 한눈에 파악
```

## 3. 타겟 사용자 (Target Audience)

### 3.1 Primary Persona: "불안한 개미"

**프로필:**

- 25-40세, 직장인/자영업자
- 투자 경력 1-5년
- 포트폴리오 500만원~5000만원
- 매일 1-2회 주가 확인

**Pain Points:**

- "주식 앱 켜기가 무서워요"
- "뉴스 볼 때마다 마음이 흔들려요"
- "남들 수익 인증 보면 조급해져요"
- "언제 팔아야 할지 모르겠어요"

**Goals:**

- 흔들리지 않는 투자 기준 갖기
- 투자에 대한 확신과 평온함
- 논리적으로 설명할 수 있는 포트폴리오

### 3.2 Secondary Persona: "성장하는 투자자"

**프로필:**

- 30-45세
- 투자 공부를 하고 있음
- 기업 분석 시도하지만 체계가 없음
- 유튜브/책으로 학습 중

**Pain Points:**

- "공부는 하는데 실전에서 적용이 안 돼요"
- "내 분석이 맞는지 확인할 방법이 없어요"
- "체계적인 투자 프로세스가 필요해요"

### 섹션 타이틀

```
"혹시 이런 경험 있으신가요?"
```

### 공감 포인트 (카드 형식)

**카드 1: 패닉셀**

```
📉 "갑자기 5% 빠졌는데..."
   → 뉴스 검색, 커뮤니티 확인
   → 불안해서 일단 팔았다
   → 다음 날 10% 반등
   → 후회와 자책
```

**카드 2: FOMO 매수**

```
📈 "다들 이거 산다던데..."
   → 급하게 매수
   → 왜 샀는지 설명 못함
   → 떨어지면 버틸 이유가 없음
```

**카드 3: 정보 과잉**

```
📰 "이 뉴스가 좋은 건가?"
   → 긍정/부정 해석 다 있음
   → 내 포지션에 어떤 의미인지 모름
   → 결국 감으로 판단
```

**카드 4: 출구 전략 부재**

```
❓ "언제 팔아야 하지?"
   → 오르면 더 오를 것 같고
   → 떨어지면 본전 올 것 같고
   → 판단 기준이 없음
```

### 섹션 마무리 카피

```
"문제는 '실력'이 아닙니다.
 문제는 '기준'이 없는 것입니다."
```

## Section 4: Features (핵심 기능)

### Feature 1: 투자 내러티브 (Narrative)

**타이틀:** "왜 이 주식인가요?"

**설명:**

```
모든 투자는 질문에서 시작합니다.

"JYP의 K-POP 시스템 수출은 성공할까?"
"Nvidia의 AI 반도체 독점은 지속될까?"

Hypo는 각 종목에 대한 핵심 투자 질문을 제시하고,
당신만의 답변을 기록하게 합니다.

이제 '왜 샀는지' 까먹을 일이 없습니다.
```

**화면:** NarrativeIntro 스크린샷

- 투자 질문 카드
- Why Important 섹션
- Current Situation 섹션
- Answer Choices

---

### Feature 2: 워치포인트 (Watchpoints)

**타이틀:** "언제 팔아야 하나요?"

**설명:**

```
가설이 맞는지 확인하는 체크포인트.

"VCHA 미국 걸그룹 성공 여부" → Bull/Bear
"콘서트 매출이 앨범 감소를 상쇄하는가" → Bull/Bear

각 워치포인트마다 당신의 기준을 설정하세요.
이 기준이 무너지면, 그때 매도를 고민하면 됩니다.

감으로 파는 게 아니라, 기준으로 파는 겁니다.
```

**화면:** WatchpointBuilder 스크린샷

- Watchpoint 질문
- Bull/Bear 선택지
- 진행 프로그레스 바

---

### Feature 3: 이벤트 대응 (Event Response)

**타이틀:** "이 뉴스, 어떻게 해야 하죠?"

**설명:**

```
중요한 이벤트가 발생하면 Hypo가 분석합니다:

✓ 이 이벤트가 내 워치포인트에 어떤 영향을 주는지
✓ 가설이 강화되는지, 약화되는지
✓ 지금 어떤 행동이 합리적인지

더 이상 뉴스에 휘둘리지 마세요.
당신의 기준에 비추어 판단하세요.
```

**화면:** ImmersiveEventMode 스크린샷

- 이벤트 상세 정보
- 워치포인트 체크 결과
- Buy/Hold/Sell 시나리오

---

### Feature 4: 로직 건강도 (Logic Health)

**타이틀:** "내 투자, 지금 괜찮은 거야?"

**설명:**

```
Hypo는 당신의 투자 가설 건강도를 0-100점으로 보여줍니다.

80점 이상: 🟢 가설 건재, 안심하고 홀드
50-79점:  🟡 주의 필요, 워치포인트 확인
50점 미만: 🔴 가설 위험, 재검토 필요

밤에 불안할 때, 앱을 열어보세요.
"로직 건강도 85점"이면 푹 주무셔도 됩니다.
```

**화면:** StockDetailModal의 Logic Health Meter

- 점수 게이지
- 상태 인디케이터
- 히스토리 변화

## Section 5: Demo (시나리오 데모)

### 섹션 타이틀

```
"JYP 투자자의 하루"
```

### 스토리텔링 형식

**Morning: 가설 설정**

```
"JYP의 K-POP 시스템 수출이 성공하면,
 현재 밸류에이션은 매력적이다."

→ 워치포인트 설정:
  1. VCHA 미국 걸그룹 성과
  2. 콘서트 매출 방어력
  3. 신인 그룹 흥행 여부
```

**Afternoon: 이벤트 발생**

```
🔔 "VCHA 글로벌 데뷔 초기 지표 발표"

Hypo 분석:
- 워치포인트 #1 관련 이벤트
- 초기 스트리밍 지표 기대 이하
- 가설 약화 가능성 있음

시나리오:
A) 장기 관점 유지 → Hold
B) 리스크 관리 → 일부 매도
C) 추가 관찰 필요 → Watch
```

**Evening: 판단**

```
"나는 B 선택: 30% 매도"

→ 로직 건강도: 85점 → 65점
→ 이유 기록: "초기 지표 부진, 리스크 관리"
→ 남은 포지션은 다음 워치포인트까지 홀드
```

**Result: 평온한 밤**

```
"내 기준대로 대응했으니,
 주가가 어떻게 되든 후회 없어."

→ 감정적 매매 ❌
→ 논리적 대응 ✅
→ 기록으로 남김 ✅
```

## Section 7: FAQ

### 자주 묻는 질문

**Q: Hypo는 종목 추천 서비스인가요?**

```
A: 아닙니다. Hypo는 종목을 추천하지 않습니다.
   대신, 당신이 선택한 종목에 대해
   투자 가설을 세우고 검증하는 것을 도와줍니다.
   투자 결정은 항상 본인의 판단입니다.
```

**Q: 초보자도 사용할 수 있나요?**

```
A: 네, 오히려 초보자에게 더 필요합니다.
   Hypo는 투자 질문과 워치포인트를 제시해주기 때문에
   투자 경험이 적어도 체계적인 사고를 할 수 있습니다.
```

**Q: 어떤 종목을 지원하나요?**

```
A: 베타 버전에서는 국내 주요 종목과
   미국 대형주를 지원합니다.
   정식 출시 시 지원 범위를 확대할 예정입니다.
```

## 11. 최종 권장 사항

### 핵심 메시지 우선순위

1. **감정적 공감**: 불안함을 건드려 주목 획득
2. **문제 정의**: "기준의 부재"가 본질임을 인식시킴
3. **해결책 제시**: 가설-검증-대응 프레임워크
4. **기능 증명**: 실제 화면과 시나리오로 신뢰 구축
5. **행동 유도**: 강력한 CTA와 인센티브

### 차별화 포인트

```
❌ "AI가 종목을 추천합니다" (경쟁 포화)
❌ "수익률 00% 달성" (신뢰도 낮음)
❌ "전문가 리포트 제공" (차별화 부족)

✅ "당신의 투자 가설을 세워드립니다" (유니크)
✅ "논리 건강도로 포지션 상태 확인" (구체적)
✅ "이벤트 발생 시 행동 가이드 제공" (실용적)
```

### A/B 테스트 우선순위

1. 헤드라인 (문제 중심 vs 해결책 중심)
2. CTA 버튼 텍스트
3. Hero 비주얼 (앱 목업 vs 일러스트)
4. Problem 섹션 길이 (상세 vs 간략)
